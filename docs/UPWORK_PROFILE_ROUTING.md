## Core strategy

### Waseem

Use as the default profile for exact AI opportunities. The public profile currently has the strongest verified credibility among the supplied profiles, so it should not be diluted with basic website or unrelated development bids.

Prioritize:

- RAG and private knowledge assistants;
- LLM and Generative AI applications;
- custom GPTs and chatbots;
- voice AI;
- machine learning and computer vision;
- document/PDF intelligence;
- AI automation with a real implementation scope.

### US full-stack profile

This is the intended route for full-stack AI and SaaS application work, but its live public profile could not be validated during the initial audit. Keep it behind human approval until the owner, true location, public title, rate, work history, Job Success Score, and Upwork verification are confirmed.

Never use this profile to override a client requirement involving location, citizenship, clearance, onsite presence, or employment eligibility.

### Roshana

Use selectively for visual-production-led jobs where the deliverable is clear and the profile has exact proof. Do not route coding-heavy browser games or broad multiplayer development here.

Before bidding, confirm:

- reference quality/style;
- exact asset count and duration;
- revision limits;
- rendering requirements;
- source-file requirements;
- approval milestones.

### Nadir

Use only for exact immersive-development matches. Strong candidates include Unity/Unreal, ARKit/ARCore, WebAR, mobile AR, VR, Meta Quest, filters, and interactive 3D.

Avoid generic animation retainers and broad multiplayer-game builds. Require a realistic budget, clear technical scope, and a directly relevant case study.

## Connect-spend rules

1. Bid only when the recommended profile has an exact service match.
2. Hot/urgent leads still require profile eligibility and scope review.
3. A profile-specific minimum score is a floor, not a reason to bid automatically.
4. Rejected, unclear, low-budget, unrealistic, or restricted jobs should not consume Connects.
5. The proposal must use evidence that belongs to, or is approved for, the selected profile.
6. All proposals remain manually reviewed and submitted.

## Re-audit triggers

Review the registry whenever any of these changes:

- Job Success Score;
- public rate;
- profile title or specialization;
- a major new review;
- profile location or identity verification;
- portfolio or work-history additions;
- profile availability or business priorities.
